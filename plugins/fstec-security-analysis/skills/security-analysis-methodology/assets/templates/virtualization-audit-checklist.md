# Чек-лист безопасности виртуализации
- Гипервизор: доверенная версия, список разрешённых модулей/драйверов, патчи/обновления, отключение ненужных сервисов.
- Изоляция VM/тенантов: CPU pinning/NUMA изоляция, запрет межтенантного трафика, контроль L2/L3, защита от side-channel (LLC/SMT/NUMA policy).
- Доступ/управление: MFA для админов, RBAC/ABAC, audit консоль/API, change management, break-glass процедуры.
- Сеть: сегментация mgmt/tenant/storage, шифрование каналов (IPsec/ГОСТ), фильтрация egress, DHCP/DNS защита.
- Хранилища/образы: шифрование VM images/backup, контроль снапшотов/клонов, подпись образов, хранилище в доверенной зоне.
- Миграции/HA: защищённые live-migration каналы, контроль целостности, запрет миграций между недоверенными хостами.
- Логи/мониторинг: аудит гипервизора/админов, SIEM, time sync, защита логов, ретеншн.
- СЗИ/сертификация: перечень сертифицированных средств (СОВ/СЗИ НСД/МЭ), актуальные сертификаты, совместимость с RedOS 7/8.
- Аттестация: матрица контролей (17/21/118), evidence pack (конфиги, логи, тесты), статус remediation.

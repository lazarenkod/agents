# Требования к защите среды виртуализации (ФСТЭК ориентиры)
- Нормативка: приказы 17/21, методические материалы по защите виртуализации, увязка с 118 при использовании контейнеров внутри VM.
- Гипервизор: доверенные версии, контроль загрузки (Secure Boot/TPM), ограничение модулей/драйверов, запрет kexec, аудит изменений.
- Изоляция: многоуровневая сегментация mgmt/tenant/storage, защита от L2/L3/side-channel, NUMA/SMT политика, ограничения по shared ресурсов (hugepages, PCI passthrough).
- Управление доступом: MFA, RBAC/ABAC, отдельные домены администрирования, журналирование действий, регулярный пересмотр привилегий.
- Сеть/каналы: шифрование трафика администрирования/миграций, VPN/IPsec/ГОСТ, контроль egress, DDoS защита, доверенные DNS/NTP.
- Хранилища/образы: шифрование VM/backup, подпись образов, контроль снапшотов/клонов, политика уничтожения, контроль доступа к datastore.
- Миграции/HA: защищённые live migration, запрет межконтурных миграций без аттестации, контроль целостности и идентичности целевых хостов.
- Логи/мониторинг: централизованный сбор с гипервизоров/управляющих узлов, защита логов, SIEM, синхронизация времени, алерты на админские действия/конфиг-изменения.
- СЗИ: перечень сертифицированных средств (СОВ/СЗИ НСД/МЭ), актуальные сертификаты, подтверждение совместимости с выбранной ОС (RedOS 7/8 и др.).
- Аттестация: модель угроз для виртуализации, матрица контролей, программа/методика испытаний, evidence pack, регламент изменений после аттестации.

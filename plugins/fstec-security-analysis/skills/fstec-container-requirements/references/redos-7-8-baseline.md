# RedOS 7/8: baseline и сведения о сертификации

## Где брать официальную информацию
- Реестр сертификатов ФСТЭК + страница RedOS (релиз/сертификат) → фиксируйте номер сертификата, класс СВТ, область применения, срок действия, протоколы испытаний.
- Выписки из сертификата: версии ядра, криптомодули, состав пакетов, условия применения (например, запрещенные сервисы/модули).
- Требование: хранить копии сертификатов и протоколов, вести журнал изменений, влияющих на область сертификации.

## Типовые сертифицированные составы (заполняйте по актуальной справке)

| Версия RedOS | Класс СВТ (ОС) | Базовое ядро | Крипто/СКЗИ | Ключевые пакеты | Ограничения/условия |
|---|---|---|---|---|---|
| 7.x (пример) | 4 класс | 4.9/4.14 LTS (уточнить) | КриптоПро, ViPNet (по сертификату) | SELinux, auditd, systemd 219+, firewalld/nftables, openssl 1.0.2/1.1 | Запрет сторонних repo, отключить kexec, список допустимых модулей ядра |
| 8.x (пример) | 3/4 класс | 5.10 LTS (уточнить) | КриптоПро, ViPNet (по сертификату) | SELinux, auditd, systemd 250+, nftables, openssl 1.1.1/3.0, podman/docker поддержка | Только сертифицированные repo, контроль хэшей, запрет незаявленных модулей |

> Примечание: замените «пример» на точные данные из актуального сертификата ФСТЭК и паспорта версии RedOS. Для аудита фиксируйте контрольные суммы iso/репозиториев.

## Базовые требования
- Репозитории: только доверенные сертифицированные, контроль подписей/хэшей, запрет сторонних rpm; собственные зеркала с регулярной сверкой.
- Ядро/SELinux: enforcing, auditd, kexec disabled, sysctl hardening; проверка совместимости kubelet/containerd/docker/podman с сертифицированным ядром.
- Пользователи/доступ: MFA для админов, запрет shared accounts, sudo по заявкам, раз в квартал ревью учёток.
- Логи/аудит: auditd + централизованный сбор, контроль целостности, запрет локальной очистки, ретеншн ≥ 1 год (или согласно политике).
- Криптография: использовать сертифицированные СКЗИ/библиотеки, TLS профили, управление ключами, проверка CRL/OCSP; для ГОСТ — соответствующие пакеты из сертифицированного состава.
- Обновления: только из доверенных репозиториев, тестовый контур, контроль хэшей, регламент на обновления и откат; ведите журнал применённых патчей и влияние на область сертификации.

# DR Runbook Template

## Контекст
- Сервис/данные, критичность, RTO/RPO, регуляторика

## Триггеры
- Условия активации (инциденты/регион/az/данные)

## Шаги восстановления
- Шаги с командами/скриптами/ссылками, владельцы, ETA
- Данные: источники бэкапов/репликаций, проверки целостности
- Инфра: переключение, DNS, конфиги, секреты

## Проверки и валидация
- Тесты, критерии успешного восстановления, кто подтверждает

## Коммуникации
- Кто уведомляется, каналы, частота, шаблоны сообщений

## Пост-инцидент
- RCA, обновление артефактов, улучшения

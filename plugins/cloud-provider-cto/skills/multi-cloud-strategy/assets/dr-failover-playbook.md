# Multi-Cloud DR/Failover Playbook

## Сценарий
- Какой паттерн (active-active / primary-DR), триггеры, RTO/RPO

## Шаги
- Детект/алёрт, изоляция, переключение (DNS/GSLB/traffic manager), данные/консистентность, конфиги/секреты

## Проверка
- Валидация сервисов, метрики (latency/ошибки), обратная связь клиентов

## Коммуникации
- Внутренние/внешние каналы, частота апдейтов, роли

## Пост-инцидент
- RCA, улучшения, обновление артефактов

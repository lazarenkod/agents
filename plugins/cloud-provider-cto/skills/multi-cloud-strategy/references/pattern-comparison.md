# Сравнение паттернов multi-cloud
- **Active-Active:** высокая доступность/latency, сложность и стоимость; требует согласованности данных и унифицированного IAM/obs.
- **Primary/DR:** дешевле, проще операции; egress/время восстановления; регулярные тесты.
- **Workload Split (per service/region):** снижает lock-in/стоимость, но увеличивает операционную сложность и дрейф конфигов.
- **Vendor Diversity (best-of-breed):** использовать уникальные сервисы; риски интеграции/данных/поддержки.

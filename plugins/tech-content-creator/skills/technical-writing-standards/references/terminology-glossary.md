# Глоссарий технических терминов / Technical Terminology Glossary

Справочник по переводу и использованию технических терминов в русском и английском языках.

---

## Принципы использования терминов / Terminology Usage Principles

### Когда использовать заимствование / When to Borrow

✅ **Используйте английский термин, если:**
- Термин устоялся в русскоязычном сообществе (Git, Docker, API)
- Русский перевод звучит неестественно или создает путаницу
- Термин является брендом или продуктом (Kubernetes, PostgreSQL)
- В профессиональном контексте термин используется на английском

### Когда переводить / When to Translate

✅ **Переводите термин, если:**
- Существует точный и устоявшийся русский эквивалент
- Термин используется в деловом/нетехническом контексте
- Аудитория включает нетехнических специалистов
- Документация предназначена для широкой аудитории

## A

### API (Application Programming Interface)
**Русский**: API (не переводится), интерфейс программирования приложений
**Использование**: Всегда используйте "API" в технических текстах
**Пример**:
- EN: "The REST API returns JSON responses"
- RU: "REST API возвращает ответы в формате JSON"

### Architecture / Архитектура
**Русский**: Архитектура
**Использование**: Прямой перевод
**Термины**:
- Microservices architecture — Микросервисная архитектура
- Monolithic architecture — Монолитная архитектура
- Event-driven architecture — Событийно-ориентированная архитектура
- Serverless architecture — Бессерверная архитектура

### Authentication / Авторизация
**Русский**: Аутентификация (проверка подлинности), Авторизация (проверка прав)
**Важно**: Часто путают!
- **Authentication (Аутентификация)**: Кто вы? (проверка учетных данных)
- **Authorization (Авторизация)**: Что вам можно? (проверка прав доступа)

## B

### Backend / Бэкенд
**Русский**: Бэкенд, серверная часть
**Использование**: "Бэкенд" в неформальных текстах, "серверная часть" в формальных
**Пример**:
- Habr: "Наш бэкенд построен на Node.js"
- RBC: "Серверная часть системы обрабатывает запросы"

### Backup / Резервная копия
**Русский**: Резервная копия, бэкап
**Использование**: "Резервная копия" в формальных текстах, "бэкап" допустим в технических
**Термины**:
- Backup strategy — Стратегия резервного копирования
- Full backup — Полная резервная копия
- Incremental backup — Инкрементная резервная копия

### Bug / Баг
**Русский**: Ошибка, баг, дефект
**Использование**:
- Формальный: "ошибка в коде"
- Технический: "баг" (широко принято)
- Деловой: "дефект программного обеспечения"

## C

### Cache / Кэш
**Русский**: Кэш
**Использование**: Используйте "кэш" во всех контекстах
**Термины**:
- In-memory cache — Кэш в оперативной памяти
- Cache invalidation — Инвалидация кэша
- CDN (Content Delivery Network) — CDN, сеть доставки контента

### CI/CD (Continuous Integration/Continuous Delivery)
**Русский**: CI/CD, непрерывная интеграция и непрерывная доставка
**Использование**: Используйте "CI/CD" в технических текстах
**Термины**:
- CI/CD pipeline — CI/CD-конвейер, пайплайн CI/CD
- Build pipeline — Конвейер сборки

### Cloud / Облако
**Русский**: Облако, облачная среда, облачная инфраструктура
**Использование**: Зависит от контекста
**Термины**:

- Cloud-native — Облачно-ориентированный, cloud-native (допустимо)
- Multi-cloud — Мультиоблачная среда

### Container / Контейнер
**Русский**: Контейнер
**Использование**: Всегда "контейнер"

- Container orchestration — Оркестрация контейнеров

## Сокращения и аббревиатуры / Acronyms and Abbreviations

### Правила использования / Usage Rules

1. **При первом упоминании**: Полная форма + аббревиатура в скобках
   - EN: "Continuous Integration (CI)"
   - RU: "Непрерывная интеграция (CI)"

2. **Далее**: Только аббревиатура

3. **Международные аббревиатуры**: Обычно не переводятся
   - API, REST, HTTP, TCP/IP, JSON, XML

## Стилистические рекомендации / Style Recommendations

### Смешанное использование / Mixed Usage

**Допустимо**:
- "REST API" (смесь английского термина и аббревиатуры)
- "CI/CD-пайплайн" (аббревиатура + транслитерация)
- "Kubernetes-кластер" (бренд + русское слово)

**Следите за консистентностью**:
- Если начали с "пайплайн", используйте везде "пайплайн" (не чередуйте с "конвейер")
- Если выбрали "бэкенд", не переключайтесь на "серверную часть" в том же тексте

### Контекстная адаптация / Context Adaptation

**Для разработчиков** (Habr, VC.ru):
- Больше заимствований допустимо
- "деплой", "пайплайн", "воркфлоу" приемлемы
- Технический сленг понятен

**Для бизнес-аудитории** (RBC, Vedomosti):
- Больше переводов
- "развертывание", "конвейер", "рабочий процесс"
- Формальный стиль

**Для международной аудитории**:
- Используйте устоявшиеся английские термины
- Избегайте русских неологизмов
- Следуйте industry standards

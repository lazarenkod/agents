# Политика cost-aware роутинга
- Сегментация: путь/фича/тенант, SLA по качеству/latency.
- Маршруты: дешёвая модель по умолчанию, upscale при сложных запросах/low confidence, fallback при ошибках/квотах.
- Ограничения: max tokens, banned features, PII/регион.
- Фин-контроль: квоты $/день, алерты на рост $/req, трекинг экономии.
- Управление: фич-флаги, канареечный rollout, аудит изменений.

# Data Refresh Runbook (RAG)

## Триггеры
- Частота обновлений, события (релиз, новые данные, регуляторные изменения)

## Шаги
- Получение данных → нормализация → chunking → индексация/репликация → валидация → rollout
- Проверки: размер/версии, PII/редакция, консистентность, деградации индекса

## Валидация
- Сравнение метрик (search/latency/quality), smoke тесты, regression eval

## Rollback/DR
- Как откатить индекс/модель/кеш, кого уведомить, алерты

## Коммуникации
- Каналы, владельцы, расписание, пост-обновление отчёт

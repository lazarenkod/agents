# Политика роутинга моделей
- Цели: минимальный латентность/стоимость при соблюдении качества/безопасности.
- Правила: сегменты трафика (premium/стандарт), лимиты токенов, языки/страны, PII/регуляторика.
- Маршруты: baseline → cache → light model → heavy model; fallback цепочки.
- Мониторинг: success/quality rate, latency/cost per route, override логи.
- A/B: эксперименты по выбору модели, минимальная выборка, критерии остановки.

# Chaos Test Plan

## Цель
- Проверить устойчивость к отказам (сеть/узлы/сервисы/регион)

## Сценарии
- Drop/latency сети, падение узлов/брокеров/БД, перегруз очередей, обрыв region/AZ

## Метрики
- Latency/error, failover время, потеря данных, деградация throughput, алерты

## Критерии
- Успех/фейл, пороги стопа, rollback, уведомления

## План улучшений
- Найденные слабые места, действия, владельцы/сроки

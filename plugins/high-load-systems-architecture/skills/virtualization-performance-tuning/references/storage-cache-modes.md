# Кэш режимы QEMU
- `none`: прямой I/O, min latency, требует надежного хранилища; лучше для NVMe/RAID.
- `writethrough`: безопасно, запись через хост кеш → диск, латентность выше.
- `writeback`: быстро, но риск при сбое; рекомендуется с UPS/журналом.
- `unsafe`: максимум скорости, высокие риски потери данных.
- Тесты: fio с flush, сравнить p99 latency/throughput, учесть fsync частоту приложения.
